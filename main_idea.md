Этот проект создан для реализации небольшого web приложения написанного на Typescript

## Основной функционал приложения:
Есть набор json файлов внутри у которых собрано информация для проведения интервью на должность разработчика
структура этого json описана в GUIDE_JSON_SCHEMA.md

внутри json многие блоки содержат в себе текст в формате markdown нужно реализовать что бы приложение выводило их на экран с сохранением формата

## как я вижу UI приложения:
Есть главная страница на которой ты имеешь возможность выбрать на какую должность тебя будет экзаменовать как интервьюер приложение каждой должности соответствует свой отдельный json файл из директории jsons
После выбора должности тебе тебе выводятся обзорные данные об этом интервью из Мета данных внутри json, А также оглавление при выборе пункта которого ты можешь перейти к конкретному вопросу, 
  что делать при открытии конкретного вопроса обсудим ниже. на каждом пункте оглавления должна отображаться соответсвующая статстика (бал должности, рублики, вопроса)

Также ты можешь не выбирать конкретно вопрос о приступить в целом к интервью (кнопка "приступить к интервью").

из вопросов должна быть возможность вернуться на предыдущий шаг. 
со Страницы обзорных данных должности должна быть возможность вернуться к выбору одной из должностей.


## Как я вижу процесс интервью:
На экран рандомно выводятся вопросы из выбранной должности с наименее высоким рейтингом качества ответов (позже будет описание методики сбора статистики по ответам интервью).
так же под вопросом есть кнопка "показать ответ"

после нажатия "показать ответ" раскрывается блок с ответом, а под ним блок оценки где пользователь вводит балл (подумать над интерфейсом..) бал может быть от 0 - совсем не знает ответ, до 5 - полный ответ
этот бал вносится в статистику (приязывается бал к самому вопросу и пересчитывается балл для guide_chapter и всей должности в целом)

## методика сбора статистики по вопросам интервью:
В момент открытия должности на главном экране проверяется а есть ли для данной должности какая бы не была статистика,
если статистики нет то к каждому вопросу внутри внутри связанного файла присваивается значение 0, то есть минимальный рейтинг качества ответов

## предварительная структура хранения статистики для должности (при необходимости доработай структуру):
{
должность,
связанный исходный json файл
общий бал для должности
statistic:
  {
    chapter_title
    chapter_number
    общий бал для вопросов в guide_chapter
    questions:
      {
        question_title
        question_number
        бал ответа
      }
  }
}

статистика хранится на клиенте в localstorage (для каждой должности отдельно)
